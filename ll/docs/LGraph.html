Introduction

The component LGraph provides support for representing graphs, both directed and undirected. This component is based upon the idea that it have to represent only the topological information of a graph. Basically it keeps track only of the connections between the nodes. Adding info to edges and nodes, is demanded to others components such as LEdgeInfo and LNodeInfo; we strongly recommend to look also at theirs documentation.

API Reference

Macro Arguments Description
LGraph_GetNextNode(v) A node "v". Gives the node next to "v" in the graph.
LGraph_ForAllNodes(v,g) A node "v", a graph "g". Each iteration assigns the next node in the graph to "v".
NOTE: "v" must not be deleted during the loop.
LGraph_GetNextEdge(e) An edge "e". Gives the edge next to "e" in the graph.
LGraph_ForAllEdges(e,g) An edge "e", a graph "g". Each iteration assigns the next edge in the graph to "e".
NOTE: "e" must not be deleted during the loop.
LGraph_GetSource(e) An edge "e". Gives the source node of an edge e.
NOTE: On undirected graphs the edge Source("A")->Target("B") means that from "B" you can reach also "A"; this should be taken into consideration when writing algorithms on graph visits.
LGraph_GetTarget(e) An edge "e". Gives the target node of an edge e.
See also the note in LGraph_GetSource(e)
LGraph_GetFirstOutEdge(v) A node "v". Gives the first egde in the outlist of a node "v".
LGraph_GetNextOutEdge(v,e) A node "v", an edge "e". Gives the edge next to "e" in the outlist of a node "v".
LGraph_ForAllOut(v,e) A node "v", an edge "e". Each iteration assigns the next edge in the outlist of "v" to "e".
NOTE: "e" must not be deleted during the loop.
LGraph_GetFirstInEdge(v) A node "v". Gives the first egde in the inlist of a node "v".
LGraph_GetNextInEdge(v,e) A node "v", an edge "e". Gives the egde next to "e" in the inlist of a node "v".
LGraph_ForAllIn(v,e) A node "v", an edge "e". Each iteration assigns the next edge in the inlist of "v" to "e". This macro has meaning only if the graph is directed.
